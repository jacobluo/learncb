## 产品概述

一个本机单人使用的 React TODO 应用，采用奶油风设计搭配橙色点缀，前后端分离架构，数据持久化到 SQLite。

## 核心功能

- 添加新待办事项（输入文本，回车或点击按钮添加）
- 删除待办事项
- 切换待办事项完成状态（点击勾选）
- 按「全部 / 进行中 / 已完成」三种状态过滤显示
- 数据持久化到 SQLite（重启不丢失）
- 一条 `npm run dev` 命令同时启动前后端

## 技术栈

- 前端：Vite + React + TypeScript + Tailwind CSS
- 后端：Express + better-sqlite3 + TypeScript（tsup 构建）
- 并发启动：concurrently
- 数据库：SQLite（todo.db 文件存于项目根目录）

## 实现方案

### 整体架构

采用 monorepo 结构，根目录统一管理依赖和启动脚本。前端 Vite dev server（5173 端口）通过 proxy 转发 API 请求到后端 Express（3001 端口），避免跨域问题。后端使用 better-sqlite3 同步操作 SQLite，简单可靠。

### 数据流

用户操作 → React 组件 → fetch API → Express 路由 → better-sqlite3 → SQLite 文件

### API 设计

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | /api/todos | 获取全部待办 |
| POST | /api/todos | 新增待办 |
| PATCH | /api/todos/:id | 更新待办（切换完成状态） |
| DELETE | /api/todos/:id | 删除待办 |


### 数据库表

```sql
CREATE TABLE todos (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  text TEXT NOT NULL,
  completed INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now'))
);
```

### 关键技术决策

- **better-sqlite3 而非异步 ORM**：本机单人场景，同步 API 更简单、零回调地狱、性能足够
- **Vite proxy 而非 CORS**：开发体验更好，无跨域问题
- **concurrently 同时启动**：一条命令搞定，符合交付要求
- **Tailwind CSS**：快速实现奶油风+橙色点缀的自定义设计
