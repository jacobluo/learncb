# 今日运势 · 幸运选择

## 玩法

1. 从词语库随机抽取 4 个词，作为 A/B/C/D 选项
2. E 选项让用户自行填写
3. 用户选择后，结合当日日期 + 选项内容，用确定性算法生成今日运势

## 执行步骤

1. 从词语库随机抽 4 个词 → A/B/C/D
2. **使用 `ask_followup_question` 工具**展示可点击的选项卡，让用户鼠标点选。调用方式：

```
ask_followup_question(
  title="🔮 今日运势 · 选择你的幸运词",
  questions='[{"id":"fortune","question":"点选幸运词，或直接在对话框输入任意词：","options":["{词1}","{词2}","{词3}","{词4}"],"multiSelect":false}]'
)
```

3. 用户点选选项卡或直接在对话框输入自定义词语后，提取词语
4. 用以下确定性算法生成运势：

```
seed = 日期字符串(YYYY-MM-DD) + 用户选择词语
hash = seed 每个字符的 charCode 求和
fortuneIndex = hash % 12
colorIndex   = (hash >> 4) % 12
luckyNum     = (hash % 9) + 1
```

5. 用 Markdown 格式输出运势结果卡片

## 提示语生成规则

根据词语类型给一句建议：

- 动物类（凤凰、麒麟、仙鹤、飞马、蝴蝶、鲸鱼、青龙、白虎、朱雀、玄武）→ 建议与自然/直觉相关
- 自然类（星辰、春风、莲花、明月、彩虹、瑞雪、竹子、银杏、朝阳、祥云）→ 建议与心态/修养相关
- 物品类（宝剑、金元宝、玉如意、福字、灯笼、烟花、帆船、茶壶、风筝、罗盘、钥匙、咖啡、吉他、滑板、相机、画笔、围棋）→ 建议与行动/创造相关

## 输出格式（严格遵循，使用 Markdown）

用户选择后，**必须严格按以下 Markdown 模板输出**，将占位符替换为实际值。不要使用 ASCII 框线，不要生成文件，直接在聊天中输出：

```
> ## 🔮 今日运势 · {YYYY}年{MM}月{DD}日
>
> **你的选择：** {词语}
>
> ### {emoji} {等级标签}
>
> {fortuneTexts[fortuneIndex] 替换后的文本}
>
> ---
>
> 💡 **幸运提示：** {根据词语类型生成的建议}
>
> 🎨 **幸运色：** {colors[colorIndex]}
>
> 🍀 **幸运数字：** {luckyNum}
```
