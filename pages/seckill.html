5.1

开发 Java 秒杀商品 API 服务

项目背景

使用 Spring Boot 开发一个秒杀商品的 API 后端服务，需要处理高并发库存扣减、订单创建、防超卖等核心问题。采用 OpenSpec（规范驱动开发）+ Superpowers（工程方法论）组合，分两个 Spec 迭代完成。

Spring Boot 3 MyBatis-Plus Redis MySQL OpenSpec Superpowers

阶段零：项目初始化

1

初始化 OpenSpec 工作目录

openspec init

在项目根目录生成 .openspec/ 目录，用于存放变更提案和技术规范。

Spec 1：数据库设计与基础模型

目标：设计秒杀业务的数据库表结构，包括商品表、秒杀活动表、秒杀订单表，生成 MyBatis-Plus 实体类和 Mapper。

Step 1 — 头脑风暴需求（Superpowers: brainstorming）

/brainstorming 秒杀商品 API 的数据库设计，需要支持：商品管理、秒杀活动配置（时间窗口、库存、限购）、秒杀订单记录、库存防超卖
🧠 需求澄清与探索：

1. 核心实体：Product（商品）、SeckillActivity（秒杀活动）、SeckillOrder（秒杀订单）
2. 关键约束：一个商品可有多个秒杀活动（不同时间段），每个活动有独立库存
3. 防超卖策略：数据库乐观锁（stock > 0）+ Redis 预扣减
4. 限购机制：user_id + activity_id 联合唯一索引
5. 建议增加：秒杀活动状态机（未开始→进行中→已结束）

是否需要支持分库分表？当前先按单库设计。

规范中的核心代码设计：

```lua -- seckill_stock.lua -- KEYS[1]: 库存 key (seckill:stock:{activityId}) -- KEYS[2]: 已购用户集合 key (seckill:bought:{activityId}) -- ARGV[1]: 用户ID -- ARGV[2]: 限购数量 -- 1. 检查用户是否已购买 local bought = redis.call('SISMEMBER', KEYS[2], ARGV[1]) if bought == 1 then return -2 -- 已购买 end -- 2. 检查库存 local stock = tonumber(redis.call('GET', KEYS[1])) if stock == nil or stock <= 0 then return -1 -- 库存不足 end -- 3. 扣减库存 + 记录用户 redis.call('DECR', KEYS[1]) redis.call('SADD', KEYS[2], ARGV[1]) return 1 -- 成功 ```
```java @Service @Slf4j public class SeckillServiceImpl implements SeckillService { @Autowired private StringRedisTemplate redisTemplate; @Autowired private SeckillActivityMapper activityMapper; @Autowired private SeckillOrderMapper orderMapper; @Autowired private RedisScript seckillScript; @Override @Transactional(rollbackFor = Exception.class) public SeckillResult executeSeckill(Long userId, Long activityId) { // 1. 活动有效性校验 SeckillActivity activity = activityMapper.selectById(activityId); if (activity == null || activity.getStatus() != 1) { throw new BizException(ErrorCode.ACTIVITY_NOT_STARTED); } // 2. Redis 预扣减（Lua 原子操作） Long result = redisTemplate.execute(seckillScript, List.of("seckill:stock:" + activityId, "seckill:bought:" + activityId), userId.toString(), activity.getBuyLimit().toString()); if (result == -1) throw new BizException(ErrorCode.STOCK_NOT_ENOUGH); if (result == -2) throw new BizException(ErrorCode.ALREADY_BOUGHT); // 3. 数据库乐观锁扣减（兜底） int rows = activityMapper.deductStock(activityId); if (rows == 0) { // Redis 回滚 redisTemplate.opsForValue().increment("seckill:stock:" + activityId); redisTemplate.opsForSet().remove("seckill:bought:" + activityId, userId.toString()); throw new BizException(ErrorCode.STOCK_NOT_ENOUGH); } // 4. 创建订单 SeckillOrder order = new SeckillOrder(); order.setUserId(userId); order.setActivityId(activityId); order.setProductId(activity.getProductId()); order.setOrderNo(SnowflakeIdGenerator.nextOrderNo()); order.setSeckillPrice(activity.getSeckillPrice()); order.setQuantity(1); order.setStatus(0); orderMapper.insert(order); return SeckillResult.success(order.getOrderNo()); } } ```
```java @RestController @RequestMapping("/api/seckill") @Slf4j public class SeckillController { @Autowired private SeckillService seckillService; @PostMapping("/{activityId}") @RateLimit(qps = 1000) // 令牌桶限流 public Result doSeckill( @PathVariable Long activityId, @RequestHeader("X-User-Id") Long userId) { log.info("秒杀请求: userId={}, activityId={}", userId, activityId); SeckillResult result = seckillService.executeSeckill(userId, activityId); return Result.ok(result); } @GetMapping("/{activityId}/result") public Result queryResult( @PathVariable Long activityId, @RequestHeader("X-User-Id") Long userId) { OrderVO vo = seckillService.querySeckillResult(userId, activityId); return Result.ok(vo); } } ```
