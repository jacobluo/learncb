3.2

配置 Skills 2-3 分钟

什么是 Skills？

Skills（技能）是封装好的专业指令集，可以让 AI 在特定领域表现更出色。例如：安全审计、性能压测、代码审查、前端开发等。当你在对话中触发 Skill 时，其内容会被注入到 AI 的上下文中，引导 AI 按照专业流程执行任务。

获取方式一览

方式说明适用场景
插件市场 ⭐ /plugin → 搜索一键安装 最快上手，社区最佳实践 Skill 包
GitHub 安装 从 GitHub 克隆到 .codebuddy/ 离线环境或需要版本控制
npm 安装 npm install -g 安装开发框架 规范驱动开发等独立 CLI 工具
手动创建 在 .codebuddy/skills/ 下自建 SKILL.md 定制团队规范、专属业务流程
方式一

插件市场一键安装 推荐

1
打开插件市场

在 CodeBuddy 对话框中输入 /plugin，或点击右上方 设置 ⚙️ → 切换到 插件 标签页。

2
搜索并安装

在插件市场搜索框中输入关键词（如 superpowers），找到目标插件后点击安装。

CodeBuddy 插件市场：搜索 superpowers 并安装
3
验证安装

安装完成后，验证 Skill 是否生效：

环境操作说明
CodeBuddy CLI 输入 /skills 查看所有已安装的技能列表
CodeBuddy IDE 输入 / 后搜索技能名 如输入 /br 可看到 brainstorming 技能
在 IDE 中输入 /br 即可看到 superpowers 的 brainstorming 技能

推荐插件：Superpowers — 工程方法论技能包

Superpowers 是社区最热门的 Skill 包之一，提供一整套软件工程方法论技能，覆盖从需求分析到代码审查的完整开发周期。支持 Claude Code、CodeBuddy、OpenCode 等多种 AI 编程工具。

Superpowers 包含的核心技能

🧠

brainstorming

创意探索与需求澄清，任何创造性工作之前的头脑风暴

📝

writing-plans

将需求拆解为结构化的实施计划，明确步骤与依赖

🧪

test-driven-dev

TDD 方法论，先写测试再写实现代码

🔍

systematic-debugging

遇到 Bug 先系统分析，而不是盲目修改

📋

code-review

完成功能后的代码审查，确保质量达标

✅

verification

声明完成之前必须运行验证，用证据说话

方式二

从 GitHub 手动安装

以 Anthropic 官方 Skills 仓库为例

anthropics/skills 是 Anthropic 官方发布的 Agent Skills 仓库（⭐ 124k），包含创意设计、开发技术、企业沟通、文档处理等多类技能，可直接克隆到本地使用。

安装步骤

⚠️ 注意：不要直接 git clone 整个仓库到 skills 目录 — 仓库包含 README、spec、template 等无关文件，且 skill 嵌套在 skills/ 子目录下，层级会不对。应只拷贝需要的 skill 文件夹。
1
克隆仓库到临时目录 
git clone https://github.com/anthropics/skills /tmp/anthropic-skills
2
拷贝需要的 skill 到目标目录 
# 全局安装（所有项目生效）— 拷贝单个 skill
cp -r /tmp/anthropic-skills/skills/pdf ~/.codebuddy/skills/pdf

# 或批量拷贝所有 skill
cp -r /tmp/anthropic-skills/skills/* ~/.codebuddy/skills/

# 项目级安装（仅当前项目）
cp -r /tmp/anthropic-skills/skills/pdf .codebuddy/skills/pdf
3
清理临时目录 
rm -rf /tmp/anthropic-skills

使用示例：frontend-design

frontend-design 是最常用的技能之一，安装后 AI 会自动生成高质量、有设计感的前端界面，避免千篇一律的"AI 风"。

1
只拷贝 frontend-design 这一个 skill 
cp -r /tmp/anthropic-skills/skills/frontend-design ~/.codebuddy/skills/
2
在对话中触发
/frontend-design 帮我创建一个深色主题的产品落地页，包含 Hero、特性介绍和定价模块
我来为你设计一个高质量的深色主题落地页...

设计方向：
- 🎨 深色背景 + 蓝紫渐变光效
- 📐 不对称布局，打破传统网格
- ✨ 滚动入场动画 + 悬停微交互

✏️ 正在生成 index.html / style.css / script.js
💡 效果：使用 frontend-design skill 后，AI 会主动选择独特的字体、大胆的配色、精致的动效，而不是生成千篇一律的模板页面。本手册的页面风格就是参考该 skill 的设计理念制作的。
⚠️ 协议提醒：文档类技能（docx/pdf/pptx/xlsx）为 source-available 协议；其他技能均为 Apache 2.0 开源协议。
方式三

npm 安装开发框架

OpenSpec — 规范驱动开发框架

OpenSpec 是一套规范驱动开发（Spec-Driven Development）框架。核心理念：在第一行代码编写之前，先与 AI 对齐需求规范，让开发过程更可预测、更可控。特别适合中大型功能开发。

与前两种方式不同，OpenSpec 是一个独立的 CLI 工具，通过 npm 全局安装，不是往 .codebuddy/skills/ 放文件。

1
安装 OpenSpec 
npm install -g openspec    # 全局安装
openspec init               # 在项目中初始化

OpenSpec 三阶段工作流

📋
阶段一：提案（Proposal）

用自然语言描述需求，AI 生成变更提案（Change Proposal），明确改什么、为什么改、影响范围

📐
阶段二：规范（Spec）→ 实施

提案通过后，AI 生成详细技术规范（函数签名、数据结构、测试用例），确认后执行编码

📦
阶段三：归档（Archive）

实施完成后归档规范，作为项目文档沉淀，后续迭代可追溯决策上下文

💡 如何选择？ 插件市场（Superpowers 等）适合快速获取工程方法论； GitHub 安装（Anthropic Skills 等）适合按需拷贝即用型能力； npm 安装（OpenSpec 等）适合引入独立开发框架； 手动创建适合定制团队专属规范。四种方式可按需组合使用。
方式四

手动创建自定义 Skill

配置步骤

1
创建目录结构

在项目根目录下创建 .codebuddy/skills/ 目录，为每个 Skill 创建一个子文件夹。

# 项目目录结构
my-project/
├── .codebuddy/
│   └── skills/
│       ├── jave-code-review/
│       │   └── SKILL.md
│       ├── api-design/
│       │   └── SKILL.md
│       └── unit-test/
│           └── SKILL.md
├── src/
└── package.json
2
编写 SKILL.md

在子文件夹中创建 SKILL.md 文件，使用 YAML 头部 + Markdown 正文定义技能行为。

```yaml --- name: java-code-review description: 专业的 Java 代码审查工具，从架构、稳定性、性能、可观测性、安全五大维度深度分析代码质量 metadata: { "version": "1.0.0", "author": "AI Code Reviewer", "category": "code-quality", "tags": ["java", "code-review", "architecture", "performance", "security"] } --- ``` # Java Code Review Skill 你是一位**资深的 Java 架构师和代码审查专家**，拥有 10+ 年的企业级 Java 开发经验。你的职责是对 Java 代码进行全面、深度的审查，从以下五个维度提供专业建议： ## 🎯 审查维度（5 大核心） ### 1. 架构合规性与边界 (Architecture & Boundaries) - **分层架构**: Controller → Service → Repository 分层是否清晰 - **职责单一**: 每个类/方法是否职责明确，符合 SRP（单一职责原则） - **依赖合理性**: - 上层不依赖下层具体实现（依赖倒置原则） - 模块间依赖关系清晰，无循环依赖 - 强依赖 vs 弱依赖（接口 vs 实现类） - **边界清晰**: DTO、VO、PO、Entity 等对象边界明确 ### 2. 稳定性 (Stability & Reliability) - **资源管理**: - 数据库连接、文件句柄、Socket 是否正确关闭 - try-with-resources 使用 - 线程池、连接池管理 - **分布式环境**: - 接口幂等性（防止重复提交） - 分布式事务处理（2PC、TCC、SAGA） - 分布式锁（Redis、Zookeeper） - **事务管理**: - @Transactional 使用是否正确 - 事务传播行为 - 事务边界 - **并发安全**: - 线程安全（synchronized、volatile、Lock） - 并发容器使用（ConcurrentHashMap、CopyOnWriteArrayList） - 避免死锁 ### 3. 性能和资源损耗 (Performance & Resource) - **数据库优化**: - SQL 是否有 N+1 问题 - 索引使用是否合理 - 批量操作 vs 单条操作 - 分页查询是否正确 - **缓存使用**: - Redis 缓存设计（过期时间、缓存穿透/击穿/雪崩） - 本地缓存（Guava Cache、Caffeine） - 缓存一致性 - **JVM 内存管理**: - 大对象创建 - 内存泄漏风险（静态集合、ThreadLocal 未清理） - 频繁 Full GC 的隐患 - 对象复用 ### 4. 可观测性和可维护性 (Observability & Maintainability) - **异常处理**: - 异常分类（业务异常 vs 系统异常） - 异常不吞没 - 自定义异常 - 全局异常处理 - **重试机制**: - 重试策略（指数退避） - 幂等性保证 - 熔断降级 - **日志打印质量**: - 日志级别正确（DEBUG/INFO/WARN/ERROR） - 关键参数记录 - 敏感信息脱敏 - TraceId 传递 - **代码可读性**: - 避免大量 if-else（策略模式、责任链模式） - 魔法数字/字符串提取为常量 - 方法长度控制（< 50 行） - 注释质量 - **扩展性**: - 开闭原则（对扩展开放，对修改关闭） - 依赖注入 vs 硬编码 - 配置外部化 ### 5. 安全合规 (Security & Compliance) - **越权防护**: - 接口权限校验 - 数据权限隔离 - 用户身份验证 - **敏感数据处理**: - 密码加密（BCrypt、Argon2） - 敏感字段脱敏 - SQL 注入防护（参数化查询） - XSS 防护 - CSRF 防护 - **防御性编程**: - 参数校验（@Valid、@NotNull） - 空指针防护（Optional、Objects.requireNonNull） - 类型安全 - 边界检查
```yaml --- name: unit-test description: 为函数/模块生成完备的单元测试用例 --- ``` # 单元测试生成技能 ## 测试策略 - 正常路径：验证预期输入的正确输出 - 边界条件：空值、极大值、极小值、空数组 - 异常路径：错误输入、网络超时、权限不足 - 并发场景：竞态条件、死锁检测 ## 技术选型 - JavaScript/TypeScript：Jest + Testing Library - Go：标准 testing 包 + testify - Python：pytest + unittest.mock ## 覆盖率要求 - 行覆盖率 ≥ 80% - 分支覆盖率 ≥ 70% - 关键路径 100% 覆盖
```yaml --- name: perf-optimize description: 分析代码性能瓶颈并给出优化方案 --- ``` # 性能优化技能 ## 分析维度 - **数据库**：慢查询、缺失索引、N+1 问题 - **前端渲染**：不必要的 re-render、大列表虚拟化 - **网络请求**：请求合并、缓存策略、CDN - **内存管理**：泄漏检测、大对象优化 ## 输出要求 1. 量化分析：给出优化前后的性能对比预估 2. 优先级排序：按性能收益从高到低排列 3. 代码示例：每个优化项附带 Before/After 代码
💡 使用方式：在对话中输入 / 或 @技能名 即可触发对应 Skill。例如输入 @code-review 审查当前文件。
