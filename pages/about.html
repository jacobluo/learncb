02CodeBuddy 介绍

三大核心交互模式

Ask

对话模式

技术问答与概念解释，不直接修改代码。适合快速获取思路或查阅 API。

示例对话：
React useEffect 和 useLayoutEffect 的区别是什么？
useEffect 在浏览器渲染完成后异步执行，而 useLayoutEffect 在 DOM 变更后、浏览器重绘前同步执行...
Craft

Agent 模式

当前上下文的代码生成与局部修改，响应迅速，适合精准函数编写或小范围重构。

示例对话：
帮我给 UserService 类添加一个分页查询方法，使用 cursor 分页
我来为 UserService 添加 cursor 分页查询方法... ✏️ 正在编辑 user.service.ts
Plan

计划模式

最强自动化模式，适用于跨文件修改或复杂功能开发。AI 先制定任务清单，确认后自主执行。

示例对话：
给项目添加完整的用户认证模块，包含注册、登录、JWT 和中间件
📋 任务计划：
1. 创建 auth.controller.ts
2. 创建 auth.service.ts
3. 配置 JWT 中间件
4. 添加路由守卫
5. 编写单元测试

进阶特性一览

🧠
Memory（记忆偏好）

自动记录偏好与项目关键背景，AI 越用越精准

📐
Rules（规则）

通过 .codebuddy/rules 强制 AI 遵循特定架构设计或编码规范

🛠️
Skills（技能库）

通过 .codebuddy/skills/ 封装专业指令集

🔌
MCP（模型上下文协议）

开放协议，支持接入第三方工具（数据库、实时搜索等）

🤖
自定义 Agent

根据特定业务逻辑或工作习惯，定制专属 AI 智能体
