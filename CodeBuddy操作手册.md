# CodeBuddy 上手操作案例

---

## 一、简介

### 1. 背景

CodeBuddy 是腾讯云推出的新一代 AI 代码编辑器，深度融合**设计、编码、调试与部署**全流程，通过高度自动化的工作流，为开发者打造丝滑的 AI 协作体验，覆盖从创意萌芽到代码上线的全生命周期。

核心能力一览：

| 能力 | 说明 |
|------|------|
| 智能对话 | 即时技术咨询，支持多轮对话与复杂逻辑推导 |
| 代码补全 | 毫秒级代码预测，深度适配项目私有风格 |
| Figma 转代码 | 一键将 UI 设计稿转化为可维护的高质量前端代码 |
| 组件库集成 | 预置企业级 UI 组件，支持自然语言调整样式与布局 |
| 智能诊断 | 自动发现潜在 Bug 与性能瓶颈，提供一键修复方案 |

### 2. 预计完成时长

| 步骤 | 预计耗时 |
|------|---------|
| 下载/安装 CodeBuddy | < 5 分钟 |
| 配置 Skills | 2-3 分钟 |
| 配置自定义智能体 | 3-5 分钟 |
| 配置 MCP | 3-5 分钟 |
| 配置 Rules | 2-3 分钟 |
| **总计** | **约 15-20 分钟** |

## 二、CodeBuddy 介绍

### 产品定位

CodeBuddy IDE 是一款**全流程 AI 代码编辑器**，提供三大核心交互模式：

| 模式 | 适用场景 |
|------|---------|
| **Ask（对话模式）** | 技术问答与概念解释，不直接修改代码，适合快速获取思路或查阅 API |
| **Craft（Agent 模式）** | 当前上下文的代码生成与局部修改，响应迅速，适合精准函数编写或小范围重构 |
| **Plan（计划模式）** | 最强自动化模式，适用于跨文件修改或复杂功能开发，AI 先制定任务清单，确认后自主执行 |

### 进阶特性

| 特性 | 说明 |
|------|------|
| **Memory（记忆偏好）** | 自动记录偏好与项目关键背景，AI 越用越精准 |
| **Rules（规则）** | 通过 `.codebuddy/rules` 强制 AI 遵循特定架构设计或编码规范 |
| **Skills（技能库）** | 通过 `.codebuddy/skills/` 封装专业指令集（如安全审计、性能压测） |
| **MCP（模型上下文协议）** | 开放协议，支持接入第三方工具（数据库、实时搜索等） |
| **自定义 Agent** | 根据特定业务逻辑或工作习惯，定制专属 AI 智能体 |

### 快捷键速查

| 操作系统 | 内联对话 | 打开侧边栏对话 |
|---------|---------|--------------|
| **macOS** | `⌘ + I` | `Ctrl + ⌘ + I` |
| **Windows** | `Ctrl + I` | `Ctrl + Win + I` |

> 💡 代码补全 Pro（NES）手动触发快捷键：`Ctrl + Shift + Enter`

## 三、操作步骤

### 3. 准备工作

#### 3.1 下载/安装 CodeBuddy（预计 < 5 分钟）

**步骤：**

1. 访问 CodeBuddy 官网：[https://www.codebuddy.ai](https://www.codebuddy.ai)
2. 根据操作系统选择对应版本下载：
   - **macOS**：下载 `.dmg` 文件，打开后将应用拖入 `Applications` 文件夹
   - **Windows**：下载 `.exe` 安装文件，双击运行安装向导
   - **Linux**：根据发行版选择 `.deb` / `.rpm` 或 `.tar.gz`
3. 启动 CodeBuddy，按提示完成**腾讯云账号登录/授权**
4. 登录成功后即可开始使用

> ⚠️ 请确保从**官方渠道**下载，保证安全性。

---

#### 3.2 配置 Skills（预计 2-3 分钟）

**什么是 Skills？**

Skills（技能）是封装好的专业指令集，可以让 AI 在特定领域表现更出色，例如安全审计、性能压测、代码审查等。

**配置步骤：**

1. 在项目根目录下创建 `.codebuddy/skills/` 目录
2. 在该目录下为每个 Skill 创建一个子文件夹
3. 在子文件夹中创建 `SKILL.md` 文件，定义技能的行为

**Skill 文件结构示例：**

```
.codebuddy/
└── skills/
    └── code-review/
        └── SKILL.md
```

**SKILL.md 示例：**

```markdown
---
name: code-review
description: 对代码进行全面审查，检查风格、安全性和性能问题
---

# 代码审查技能

## 审查范围
- 代码风格是否符合规范
- 是否存在潜在安全漏洞
- 性能优化建议
- 错误处理是否完善

## 输出格式
- 按严重级别分类（Critical / Warning / Info）
- 给出具体修改建议和代码示例
```

**使用方式：**

在对话中输入 `/` 或 `@` 引用技能名称即可触发对应 Skill。

#### 3.3 配置自定义智能体（预计 3-5 分钟）

**什么是自定义 Agent？**

自定义 Agent 允许你根据特定需求创建个性化的 AI 智能体，可以针对特定场景（如代码审查、文档生成、测试用例编写等）进行定制。

**配置步骤：**

1. 打开 CodeBuddy 侧边栏
2. 在对话面板右上方点击 **CodeBuddy Settings** 按钮
3. 找到 **自定义 Agent** 配置入口
4. 设置以下内容：

| 配置项 | 说明 |
|--------|------|
| Agent 名称 | 为智能体取一个有意义的名称 |
| System Prompt | 定义角色、能力范围和行为规则 |
| 可用工具 | 选择 Agent 可以调用的工具集 |
| 上下文范围 | 指定 Agent 关注的项目文件范围 |

**System Prompt 示例：**

```
你是一个专业的后端代码审查助手，专注于 Go 语言项目。
审查重点：
1. 并发安全问题
2. 错误处理规范
3. 资源泄露检查
4. API 接口设计合理性
回答时使用中文，给出具体的代码修改建议。
```

#### 3.4 配置 MCP（预计 3-5 分钟）

**什么是 MCP？**

MCP（Model Context Protocol，模型上下文协议）是一种标准化协议，可以让 AI 接入外部工具和数据源，大大扩展能力边界。

**配置入口：**

1. 在侧栏对话面板右上方，点击 **CodeBuddy Settings** 按钮
2. 切换到 **MCP 标签页**

**方式一：MCP Market 一键安装（推荐）**

1. 在 MCP 标签页中打开 **MCP Market**
2. 浏览可用的 MCP Server，根据需求选择
3. 点击 **一键安装**
4. 查看安装状态：
   - 🟢 绿色 → 安装成功
   - 🔴 红色 → 安装失败

**方式二：自定义配置**

1. 在 MCP 标签页下，点击 **Add MCP** 按钮
2. 在 JSON 配置文件中添加 MCP Server 配置：

```json
{
  "mcpServers": {
    "python-tools": {
      "type": "stdio",
      "command": "python",
      "args": ["-m", "my_mcp_server"],
      "env": {
        "PYTHONPATH": "/path/to/tools"
      },
      "description": "Python toolset"
    }
  }
}
```

**配置字段说明：**

| 字段 | 说明 |
|------|------|
| `type` | 通信方式，如 `stdio`（标准输入输出） |
| `command` | 启动 MCP Server 的命令 |
| `args` | 命令参数列表 |
| `env` | 环境变量配置 |
| `description` | MCP Server 的描述信息 |

**验证安装：**

安装成功后，点击 MCP Server 右侧的 **Try to Run** 按钮进行测试验证。在 Craft Agent 模式下，Agent 会自动识别并调用已安装的 MCP 工具。

#### 3.5 配置 Rules（预计 2-3 分钟）

**什么是 Rules？**

Rules（规则）为 AI 提供系统级指令和上下文信息，将编码规范、工作流程等打包在一起，确保 AI 在生成代码时遵循一致的指导原则。

> ⚠️ 规则只会添加到每个会话的开始部分，创建或修改规则后需要**新建对话会话**才能生效。

**规则层级：**

| 层级 | 存储位置 | 作用范围 | 版本控制 |
|------|---------|---------|---------|
| **项目规则** | `.codebuddy/rules` 目录 | 当前项目 | ✅ 可团队共享 |
| **用户规则** | 用户目录 | 全局，跨所有项目 | ❌ 不纳入版本控制 |

**三种规则类型：**

| 类型 | 描述 | 适用场景 |
|------|------|---------|
| **Always（总是）** | 应用于每个聊天会话 | 核心编码规范、架构约束、安全要求 |
| **Agent Requested（智能体请求）** | Agent 根据描述判断相关性后自动应用 | 文档、使用指南、参考资料 |
| **Manual（手动）** | 对话中通过 `@规则名` 提及时应用 | 特定功能开发指南、可选最佳实践 |

**创建项目规则步骤：**

1. 打开 CodeBuddy IDE 设置页
2. 在对话框顶部找到「**规则**」按钮
3. 点击「**创建规则**」→ 选择「**项目规则**」
4. 填写规则名称、类型和内容

**规则文件格式示例：**

```markdown
---
description: 项目编码规范
alwaysApply: true
enabled: true
---

# 编码规范

## 通用规则
- 使用 4 空格缩进
- 每行最大长度 100 字符
- 函数上方必须有注释

## 错误处理
- 始终使用 try-catch 包裹可能出错的代码
- 记录详细的错误日志
- 向用户显示友好的错误信息

## 安全要求
- 禁止硬编码密钥和密码
- 所有用户输入必须校验
```

**补充说明 — `CODEBUDDY.md` 文件：**

除了规则系统外，还可以在**项目根目录**放置 `CODEBUDDY.md` 文件，作为简化版的项目级规则：
- 纯 Markdown 文件，无需元数据
- 默认全文加入上下文
- 适合只需简单指令的项目

> 💡 如果根目录存在 `AGENTS.md` 而不存在 `CODEBUDDY.md`，将自动加载 `AGENTS.md`。

**最佳实践：**

| 建议 | 说明 |
|------|------|
| 控制规模 | 单条规则控制在 500 行以内 |
| 拆分组合 | 将大规则拆分为多个可组合的小规则 |
| 具体示例 | 提供具体示例或参考文件路径 |
| 合理分配 | 核心规范设为 `always`（建议 3-5 个），其他设为 `manual` 或 `requested` |

### 4. 开始 Coding

#### 4.1 模型建议

CodeBuddy 支持多种模型，根据任务复杂度选择合适的模型：

| 场景 | 推荐模式 | 说明 |
|------|---------|------|
| 简单问答 / API 查询 | **Ask 模式** | 快速获取答案，不修改代码 |
| 函数编写 / 小范围修改 | **Craft 模式** | Agent 自动生成和修改代码 |
| 跨文件重构 / 复杂功能 | **Plan 模式** | AI 先制定计划，确认后自主执行 |

**使用建议：**

- 日常开发中，**Craft 模式**是最常用的模式，可以直接在对话中描述需求
- 涉及多文件改动的复杂需求，推荐使用 **Plan 模式**，让 AI 先规划再执行
- 不确定方案时，先用 **Ask 模式**讨论思路，再切换到 Craft/Plan 执行

#### 4.2 分步骤指引

**Step 1：打开项目**

- 启动 CodeBuddy，通过 `File > Open Folder` 打开项目目录
- 或使用终端：`codebuddy /path/to/project`

**Step 2：打开对话面板**

- macOS：按 `Ctrl + ⌘ + I` 打开侧边栏对话
- Windows：按 `Ctrl + Win + I` 打开侧边栏对话
- 或点击右侧边栏的 CodeBuddy 图标

**Step 3：选择交互模式**

在对话面板顶部切换模式：
- **Ask**：纯对话问答
- **Craft**：Agent 直接操作代码
- **Plan**：先规划后执行

**Step 4：描述你的需求**

在对话框中用**自然语言**描述你想要实现的功能，例如：

```
帮我创建一个 Express 服务器，包含用户注册和登录接口，使用 JWT 认证。
```

**Step 5：审阅与确认**

- 在 **Craft 模式**下，AI 会直接生成/修改代码，你可以在 Diff 视图中审阅变更
- 在 **Plan 模式**下，AI 会先展示任务计划，确认后再执行
- 点击 ✅ 接受变更，或点击 ❌ 拒绝并重新描述需求

**Step 6：利用上下文增强效果**

在对话中可以使用 `@` 引用上下文：

| 引用方式 | 说明 |
|---------|------|
| `@文件名` | 将指定文件加入对话上下文 |
| `@文件夹` | 将整个文件夹加入上下文 |
| `@规则名` | 引用手动规则 |
| `@技能名` | 触发特定 Skill |

**Step 7：善用代码补全**

在编辑器中正常编写代码时，CodeBuddy 会自动提供：
- **Tab 补全**：按 `Tab` 键接受建议
- **NES 前瞻编辑**：按 `Ctrl + Shift + Enter` 手动触发，AI 不仅补全当前行，还会预测后续编码意图

## 四、常见问题 FAQ

| 问题 | 回答 |
|------|------|
| 规则修改后不生效？ | 需要**新建对话会话**才能加载新规则 |
| 记忆功能如何管理？ | 在对话中说"请记住/删除/更新…"，AI 会自动管理；也可在设置中全局开关 |
| MCP 安装失败？ | 检查网络连接和命令路径是否正确，尝试重新安装 |
| 如何在模式间切换？ | 在对话面板顶部直接点击 Ask / Craft / Plan 切换 |
| 代码补全不工作？ | 确认已登录账号且网络正常，检查设置中代码补全是否开启 |
